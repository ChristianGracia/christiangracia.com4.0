Dockerized + SSR

This site is built with webpack and deployed in a docker container where this application is running in a NginX web server that server side renders my angular app to users when requested.

Git CI Auto Deploy

Whenever I make a pull request or merge to the master branch, this application is built in a Docker container and deployed to my staging environment. When I push to my release branch, the application is built and deployed to production.

Responsive Global Fonts + Theming

This site has no font-size declarations in the entire application besides my screen responsive font types in my custom Angular Material theme.

I made these font types shrink or grow depending on screen size while most sites use media queries which causes font-sizes to not shrink or grow but noticeably gap up or down when break points are reached.

This site fully uses Material theming to provide a uniform look, use less code, and seamless Light/Dark mode.

Lazy Loaded Modules

Each page of this website is lazy loaded as a single module when it is requested. So when you access the home page, you haven't loaded the About or Project page yet but you reached the home page faster then you would have if you loaded all the modules.

Spotify API Integrated

This website receives my Spotify listening data through my API server. Every 24 hours this server logs into Spotify with a Google Chrome bot to get an authorizaion code which is used to get an access token.

This access token is then refreshed every 55 minutes throughout the day to provide the Spotify data shown on my website. Below, received from my API back-end, is my class file Spotify.ts which contains all this logic.

Web Analytics

When someone visits my website, a request is made to my API that emails me collected user data.
